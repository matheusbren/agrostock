Análise · Bootstrap 5.3

Componentes Utilizados

A seguir são detalhados os 10 componentes de interface do projeto AgroStock. Parte são componentes nativos do Bootstrap 5.3 (Offcanvas, Toast, Modal, Table, Form Switch e Button) e parte é estilização própria construída do zero sobre a estrutura (Badge, Progress, campos de formulário e Pagination), sinalizada com a etiqueta "Componente customizado". Para cada um são apresentados: descrição funcional, páginas de uso e trecho de código real extraído do projeto.

01 Offcanvas

Painel deslizante que emerge da lateral da tela. Utilizado como sidebar de navegação no modo mobile, substituindo a sidebar estática fixada que aparece apenas em telas ≥ 992 px (d-lg-none). Acionado pelo botão ☰ no cabeçalho, via atributos data-bs-toggle e data-bs-target.

Presente em index.html estoque.html cadastro.html detalhes.html editar.html relatorios.html configuracoes.html 
<!-- Trigger (botão ☰ no header) -->
<button data-bs-toggle="offcanvas"
        data-bs-target="#mobileSidebar">
  ☰
</button>

<!-- Painel -->
<div class="offcanvas offcanvas-start"
     id="mobileSidebar" tabindex="-1">
  <div class="offcanvas-header">...</div>
  <div class="offcanvas-body">...</div>
</div>
02 Toast

Notificação temporária (snackbar) exibida no canto inferior direito da tela. Disparada via JavaScript ao carregar as páginas de cadastro e edição, informando ao usuário que o endereço foi preenchido automaticamente via API dos Correios (CEP). Usa aria-live="assertive" para acessibilidade.

Presente em cadastro.html editar.html 
<div id="toastCEP"
     class="toast align-items-center"
     role="alert"
     aria-live="assertive"
     aria-atomic="true">
  <div class="d-flex gap-3 p-3">
    ...conteúdo...
    <button data-bs-dismiss="toast">✕</button>
  </div>
</div>

<!-- Inicialização JS -->
<script>
  const t = new bootstrap.Toast(
    document.getElementById('toastCEP'));
  t.show();
</script>
03 Modal

Diálogo sobreposto que bloqueia a interação com o restante da página enquanto ativo. Aplicado na tela de detalhes do insumo como confirmação antes da exclusão do registro, prevenindo remoções acidentais. A classe modal-dialog-centered centraliza verticalmente.

Presente em detalhes.html 
<!-- Botão que abre o modal -->
<button data-bs-toggle="modal"
        data-bs-target="#modalExcluir">
  Excluir Produto
</button>

<!-- Estrutura do Modal -->
<div class="modal fade" id="modalExcluir">
  <div class="modal-dialog modal-dialog-centered">
    <div class="modal-content">
      <div class="modal-header">...</div>
      <div class="modal-body">...</div>
      <div class="modal-footer">
        <button data-bs-dismiss="modal">Cancelar</button>
      </div>
    </div>
  </div>
</div>

Classificação Componente customizado — estilizado do zero sobre a estrutura, sem usar a classe-base do Bootstrap

Rótulo compacto inline utilizado para indicar o estado atual de cada item de estoque. Quatro variações semânticas foram implementadas com cores distintas baseadas nas variáveis CSS do tema: Em Estoque (verde), Estoque Baixo (âmbar), Em Trânsito (azul) e Esgotado (vermelho).

</span>
<span class="badge-status badge-em-transito">

</span>
<span class="badge-status badge-esgotado">

  </table>
</div>

Layout · Bootstrap 5.3

Análise de Grid e Flexbox

O AgroStock combina o sistema de Grid de 12 colunas do Bootstrap com os utilitários Flexbox para construir layouts responsivos sem escrever CSS de posicionamento manual. Cada tipo de layout foi escolhido conforme a natureza do conteúdo.

2.1 — Sistema de Grid de 12 Colunas

O Bootstrap divide cada linha (row) em 12 colunas virtuais. As colunas (col-*) definem quantas dessas 12 colunas um elemento ocupa, com variações por breakpoint (col-12, col-md-6, col-lg-3 etc.). No AgroStock, o Grid foi usado principalmente em dois contextos:

  • Cards KPI do Dashboard (index.html): Quatro métricas dispostas como col-12 col-md-6 col-lg-3 — empilhadas no mobile, 2 colunas no tablet, 4 no desktop.
  • Formulários assimétricos (cadastro.html, editar.html): Layout de col-12 col-lg-8 para o formulário principal e col-12 col-lg-4 para o painel lateral de resumo, criando proporção 2:1.
<!-- Grid KPI: 4 colunas no desktop, 2 no tablet, 1 no mobile -->
<div class="row g-4">
  <div class="col-12 col-md-6 col-lg-3">
    <div class="kpi-card">Total em Estoque</div>
  </div>
  <!-- Repetido 3x para os outros KPIs -->
</div>

<!-- Layout assimétrico do formulário -->
<div class="row g-4">
  <div class="col-12 col-lg-8">
    <!-- Seções do formulário -->
  </div>
  <div class="col-12 col-lg-4">
    <!-- Painel de resumo lateral -->
  </div>
</div>

2.2 — Utilitários Flexbox

Os utilitários Flexbox do Bootstrap (d-flex, flex-*, justify-content-*, align-items-*, gap-*) foram amplamente usados para alinhar elementos dentro de cards, cabeçalhos e linhas de ação. Diferentemente do Grid (que organiza blocos maiores em colunas), o Flexbox foi aplicado em microcomposições internas:

  • Cabeçalho da página (top-header): d-flex justify-content-between align-items-center — logo à esquerda, ações à direita.
  • Linhas de ação acima das tabelas: d-flex flex-md-row gap-2 — filtros e botões dispostos horizontalmente com quebra automática no mobile via flex-wrap.
  • Items da sidebar: d-flex flex-column gap-1 — links empilhados verticalmente com espaçamento uniforme.
  • Barra de KPI da tabela: d-flex align-items-center gap-3 — ícone + barra de progresso + valor numérico na mesma linha.
<!-- Cabeçalho com logo + ações -->
<header class="top-header d-flex
         justify-content-between align-items-center
         px-4">
  <div class="d-flex align-items-center gap-3">
    <!-- Logo + título -->
  </div>
  <div class="d-flex align-items-center gap-2 ms-auto">
    <!-- Botões de ação -->
  </div>
</header>

2.3 — Layout Combinado: Grid + Flexbox

Na tela de estoque (estoque.html), os dois sistemas coexistem no mesmo nível de DOM: o Grid posiciona os blocos bento de analytics (estatísticas rápidas) e a área de tabela em colunas responsivas, enquanto o Flexbox organiza os elementos internos de cada bloco. Essa combinação é a prática recomendada pelo Bootstrap: Grid para estrutura macro, Flexbox para microalinhamentos.

Implementação · Sticky Footer

Rodapé Fixo ao Fundo da Página

O rodapé fixo (sticky footer) é um padrão de layout onde o rodapé sempre permanece na parte inferior da tela — mesmo quando o conteúdo é curto e não preenche a altura total da viewport. O AgroStock implementa esse padrão usando exclusivamente utilitários Flexbox do Bootstrap, sem JavaScript ou posicionamento absoluto.

3.1 — A Técnica: Coluna Flexível com Elemento Expansivo

A técnica funciona transformando o contêiner principal em uma coluna flex de altura mínima de 100% da viewport. O elemento <main> recebe flex-grow: 1, o que instrui o Flexbox a expandir esse elemento para preencher todo o espaço disponível restante entre o cabeçalho e o rodapé. O rodapé, sendo o último filho sem flex-grow, naturalmente ocupa apenas sua altura intrínseca — e sempre ao final da coluna.

Diagrama Visual — Estrutura de Colunas Flex
.main-wrapper · display:flex · flex-direction:column · min-height:100vh
‹header› — Cabeçalho fixo top-header
‹main› — Conteúdo da página · EXPANDE para preencher espaço vazio flex-grow-1
‹footer› — Rodapé · sempre empurrado ao fundo app-footer

3.2 — Estrutura HTML com Classes Bootstrap

As classes Bootstrap usadas são d-flex, flex-column e min-vh-100 no wrapper, e flex-grow-1 no elemento <main>. Nenhuma das classes está no rodapé — ele simplesmente "cai para o fundo" por ser o último filho da coluna flex. 

<!--
  Wrapper: coluna flex com altura mínima = 100vh
  Bootstrap classes:
    d-flex       → display: flex
    flex-column  → flex-direction: column
    min-vh-100   → min-height: 100vh
-->
<div class="main-wrapper d-flex flex-column min-vh-100">

  <header class="top-header">
    <!-- Cabeçalho sticky (position:sticky; top:0) -->
  </header>

  <main class="flex-grow-1 px-3 px-lg-5 py-4">
    <!--
      flex-grow-1 → flex-grow: 1
      Este elemento cresce para preencher TODO
      o espaço disponível entre header e footer.
    -->
  </main>

  <footer class="app-footer d-flex
    justify-content-between align-items-center">
    <!-- Sempre ao fundo — sem classe especial necessária -->
    <span>© 2024 Ecossistema Digital AgroStock</span>
    <div class="d-flex gap-4">
      <a href="#">Privacidade</a>
      <a href="#">Suporte</a>
    </div>
  </footer>

</div><!-- /main-wrapper -->

3.3 — CSS Complementar em assets/css/style.css

O Bootstrap cuida do layout vertical (coluna flex + expansão). O CSS customizado adiciona apenas o deslocamento horizontal (margin-left) necessário para que o conteúdo não fique abaixo da sidebar fixa no desktop. No mobile, a sidebar desaparece e a margem é zerada. 

/* Empurra o wrapper para além da sidebar fixa (256px) */
.main-wrapper {
  margin-left: var(--sidebar-width); /* 256px */
}

/* No mobile (< 992px): sidebar some, margem vai a zero */
@media (max-width: 991.98px) {
  .main-wrapper {
    margin-left: 0;
  }
  .sidebar-static {
    display: none !important;
  }
}

/* Estilos visuais do rodapé */
.app-footer {
  background-color: var(--agro-surface-low);
  border-top: 1px solid var(--agro-outline-variant);
  padding: 1rem 2rem;
  font-size: 0.6875rem;
  font-weight: 700;
  text-transform: uppercase;
  letter-spacing: 0.15em;
}
Resumo da técnica: Três classes Bootstrap no wrapper (d-flex flex-column min-vh-100) + uma classe no conteúdo (flex-grow-1) são suficientes para garantir que o rodapé sempre esteja ao fundo da página em qualquer dispositivo, sem uso de position: fixed (que removeria o rodapé do fluxo normal) nem de position: absolute (que causaria sobreposição com o conteúdo).
